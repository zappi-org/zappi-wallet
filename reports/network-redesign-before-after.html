앱 사용 1시간 동안 서버로 나가는 요청의 밀도 (막대 안의 세로선 하나 = 요청 한 번)

🔓

앱을 켜고 잠금을 풀 때

매일 가장 자주 일어나는 순간

예전
  • 받은 돈 편지함을 계정 생성 시점부터 전부 다시 다운로드
  • 모든 미완료 건을 민트에 하나하나 전수 질의
  • 화면 여러 곳이 같은 민트 상태를 각자 물어봄
🔴 켤 때마다 요청 수십~수백 번
지금
  • 편지함은 책갈피 이후(마지막으로 다 받은 지점)만 다운로드
  • 내 폰에 이미 있는 정보로 장부를 먼저 정리 (요청 0번)
  • 민트 상태는 창구 한 곳이 대표로 1번 확인, 모두가 공유
🟢 필요한 만큼만, 각각 1번씩
핵심: "다 다시 확인"에서 "마지막 확인 지점 이후만"으로. 책갈피는 서버가 "여기까지 전부 줬음"이라고 확실히 답했을 때만 앞으로 갑니다. 애매하면 안 움직여요(돈을 놓치는 것보다 조금 더 받는 게 나으니까).
↩️

다른 앱 쓰다가 돌아왔을 때

카톡 확인하고 3초 만에 돌아와도, 3시간 만에 돌아와도

예전
  • 3초 만에 돌아와도 미완료 건 전부를 처음부터 재확인
  • 편지함도 다시 통째로 받아옴
🔴 복귀할 때마다 잠금 해제급 폭주
지금
  • 앱이 “얼마나 자리를 비웠는지” 기억함 (1분마다 생존 신고를 기록)
  • 5분 미만 부재: 우체국 연결만 다시 붙임 (거의 0번)
  • 5분 이상 부재: 그때만 미완료 건 재확인
🟢 짧은 외출은 공짜, 긴 외출만 점검
핵심: 잠깐 나갔다 온 것과 하룻밤 자고 온 것을 구분하게 됐습니다.
🏠

홈 화면을 켜두고 기다릴 때

보낸 돈이 도착했는지, 받을 돈이 들어왔는지 기다리는 중

예전
  • 30초마다 진행 중인 건 전부를 민트에 원격 확인
  • 5건 대기 중이면 10분에 전화 100통
  • 백그라운드에 있어도 계속 돌았음
🔴 30초마다 전건 확인 전화
지금
  • 기본은 푸시: 결과가 나오면 서버가 알려줌
  • 2분마다 내 폰의 기록만 훑고, 2분 넘게 소식 없는 것만 콕 집어 1번 확인
  • 기다리는 건이 없으면 타이머 자체가 꺼짐
🟢 푸시가 정상이면 확인 전화 0번
핵심: "전부 다 30초마다"에서 "막힌 것만 2분마다"로. 확인 전화가 실패해도 진행 중인 결제를 함부로 "실패"로 찍지 않습니다(민트가 잠깐 죽었을 수도 있으니 다음 차례에 다시).
📩

누군가 나에게 돈을 보냈을 때

상대가 보내면 내 우체국(릴레이)에 편지가 도착

예전
  • 수신 대기(구독)가 그 순간 연결돼 있던 우체국에만 신청됨. 늦게 연결된 우체국의 편지는 영영 못 받음
  • 우체국 하나가 끊기면 전부 다 껐다 켜서 또 재다운로드
  • 모르는 민트의 돈은 "받을까요?" 대기표가 메모리에만 있어서, 새로고침하면 돈이 증발할 수 있었음
🔴 놓치거나, 중복 받거나, 잃거나
지금
  • 우체국이 연결되는 순간마다 구독이 자동으로 따라붙음 (늦게 붙어도 OK)
  • 끊긴 그 우체국만 조용히 다시 연결
  • 대기표를 디스크에 먼저 저장하고 나서 "확인함" 도장. 새로고침해도 대기표가 살아있고, 앱을 다시 켜면 확인 창이 다시 뜸
🟢 유실 불가능한 순서로 변경
핵심: 순서 하나가 돈을 지킵니다. "도장 먼저, 기록 나중"이면 그 사이 앱이 죽었을 때 돈이 사라지지만, "기록 먼저, 도장 나중"이면 최악의 경우에도 같은 편지를 한 번 더 받을 뿐입니다.
⌨️

보내기: 주소를 입력하는 중

라이트닝 주소(email 모양)를 한 글자씩 타이핑

예전
  • 타이핑하다 0.5초 멈출 때마다 반쯤 쓴 주소로 실제 인터넷 요청
  • a@gmail.com 입력 중이면 gmail.co 라는 엉뚱한 서버에 접속 시도
🔴 오타 상태로도 요청 발사
지금
  • 타이핑 중에는 모양 검사만 ("이메일처럼 생겼나?")
  • 실제 확인은 [다음] 버튼을 누르는 순간 딱 1번
  • 붙여넣기와 QR 스캔은 입력이 완성된 것이므로 즉시 확인 (기존 그대로)
🟢 타이핑 중 요청 0번
핵심: "입력 중"과 "입력 완료"를 구분하게 됐습니다.
✅

보내기: 확인 화면 (수수료 표시)

내 민트 A에서 내 민트 B로 옮길 때

예전
  • 직전 화면이 이미 수수료를 계산했는데(민트 왕복 4번),
  • 확인 화면이 그 결과를 버리고 처음부터 재계산 (또 4번)
🔴 확인 화면 1번에 왕복 8번
지금
  • 직전 화면의 계산 결과를 그대로 사용
  • 같은 조건(민트·금액)이면 60초간 재계산 안 함
  • 토큰 탭의 회수 수수료도 한 번 계산하면 10분 기억
🟢 왕복 8번 → 4번 → 재진입 0번
핵심: 같은 질문을 두 번 하지 않기. 덤으로 "민트를 바꿨는데 이전 민트 수수료가 계속 보이던" 표시 버그도 함께 고쳐졌습니다.
📋

토큰(보류) 탭을 열 때

보낸 토큰이 수령됐는지 목록으로 확인

예전
  • 탭을 열기만 해도 미완료 전건을 민트에 전수 질의
  • 보류 토큰 하나하나 수수료를 매번 재계산 (건당 왕복 2번)
🔴 탭 진입 = 요청 세례
지금
  • 내 폰의 기록만 정리해서 표시 (요청 0번)
  • 원격 수령 여부는 푸시와 2분 점검이 담당
  • 수수료는 캐시에서 즉시 표시
🟢 탭 진입 = 기본 요청 0번
핵심: "화면을 본다"는 행동이 네트워크 요청의 이유가 되지 않게 했습니다.
➕

새 민트를 추가(신뢰)할 때

모르는 민트에서 온 돈을 받으려고 민트를 등록하는 순간

예전
  • 전체 복구를 통째로 실행
  • "복구된 금액"은 전후 잔액 차이로 어림 (다른 입금이 겹치면 부정확)
🔴 대포로 참새 잡기 + 어림 표시
지금
  • 그 민트 앞으로 대기 중이던 토큰만 자동으로 받아줌
  • 화면의 "복구액" = 실제로 받아진 금액의 합계
  • 이미 쓰인 토큰만 대기열에서 정리, 애매한 건 남겨둠(자동 폐기 금지)
🟢 필요한 것만, 정확한 숫자로
핵심: "민트를 신뢰한다"는 행동이 곧 "이 민트의 대기 토큰을 받겠다"는 승인이므로, 딱 그 일만 합니다.
✉️

상대방에게 ecash를 보낼 때

상대의 우체국(릴레이) 주소로 편지를 부치는 순간

예전
  • 상대 우체국에 연결하면서 내 기본 우체국 목록을 통째로 교체
  • 보낸 후엔 앱이 상대 우체국만 바라봄: 내 수신이 조용히 고장
🔴 한 번 보내면 수신 라인 파손
지금
  • 상대 우체국은 “2분 임대”로 빌려서 보내고, 확인되면 자동 반납
  • 내 기본 연결과 수신 대기는 전혀 건드리지 않음
🟢 보내기와 받기가 서로 간섭 없음
핵심: 실제로 있던 버그였습니다. "잠깐 쓰는 회선"과 "내 상시 회선"을 분리하는 것이 근본 수리였어요.
🛠️

설정에서 [지갑 복구] 버튼을 누를 때

뭔가 이상해서 사용자가 직접 전체 점검을 원할 때

예전
  • 탭 진입·새로고침과 똑같은 복구가 실행됨 (전용 의미 없음)
  • 버튼 연타 시 중복 실행 위험
🔴 어디서나 같은 대포
지금
  • 이 버튼만 진짜 전수 점검 전용 (민트 전 항목 확인)
  • 연타하면 진행 중인 점검에 합류할 뿐, 두 번 돌지 않음
  • 평상시 트리거들은 가벼운 경로만 사용
🟢 대포는 사용자가 원할 때만
핵심: "전부 확인"은 사라진 게 아니라, 사용자가 명시적으로 원하는 곳 한 군데로 이사했습니다.

🚨 만약 새 방식에 문제가 생기면?

변경마다 되돌림 스위치가 있습니다. 브라우저 저장소에 값 1을 넣고 앱 잠금을 다시 풀면, 그 부분만 예전 방식으로 즉시 복귀합니다. 그리고 설정 → 환경설정 → 진단 화면에서 "푸시가 잘 오고 있는지" 숫자로 직접 확인할 수 있어요.

zappi.ks.cursor편지함 책갈피 끄기 (전체 재다운로드로 복귀)
zappi.ks.mint-info-facade민트 정보 창구 끄기 (화면별 개별 확인으로 복귀)
zappi.ks.recovery-split복구 분리 끄기 (예전처럼 전부 실행)
zappi.ks.tls-sweep2분 점검 끄기 (30초 전수 확인으로 복귀)
zappi.ks.nostr-controller우체국 교환원 끄기 (예전 연결 방식으로 복귀)
진단 화면의 합격 기준7일간 "점검이 푸시 대신 구해준 건수 = 0" 이면 새 방식이 안전하다는 증거
